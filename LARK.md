# LARK.md - 飞书交互协议

本文件只定义飞书交互相关的协议。硬规则见 RULES.md。

## 身份分工

- Bot 身份：对话入口、群聊回复、任务通知。
- 用户身份：读取已授权的个人云文档和云空间资料。

## 指令前缀

| 前缀 | 动作 |
|------|------|
| `收录：` | 创建收录任务（PDF/网页/文档/附件/聊天摘录） |
| `查：` | 查询知识库，要求给依据和限制 |
| `补证：` | 围绕某问题寻找更多来源 |
| `整理：` | 处理待入库资料 |
| `发布：` | 发布到飞书云文档（需确认） |
| `状态：` | 查看任务和知识库状态 |
| `复核：` | 检查某结论是否证据充分 |

无法识别前缀时按普通问题处理。

## 超时与回复（硬规则见 RULES.md §16-24）

- ≤10 秒：直接回复。
- >10 秒：先写 knowledge/tasks/ → **第一条用户消息 = 任务回执** → 再后台处理。

## 收录回复

```text
已创建任务：<task-id>
来源：<来源名称>
来源等级：一手 / 二手 / 待验证 + 一句话理由
处理策略：...
需要确认：...
```

## 查询回复

按 KNOWLEDGE.md 回答格式输出。飞书回复可以短，但不能省略依据和限制。

## 任务生命周期

```text
received → classified → processing → drafted → review_needed | done → done | failed
```

## 发布边界

飞书云文档只保存专题报告、综合页、状态报告。
不保存知识页、原始转换产物或高频小文件。
