# LARK.md - 飞书交互协议

本文件只定义飞书交互相关的协议。硬规则见 `RULES.md`。

## 身份分工

- Bot 身份：对话入口、群聊回复、任务通知。
- 用户身份：读取已授权的个人云文档和云空间资料。
- 群聊中不跨群携带上下文，从严处理私人资源。

## 指令前缀

| 前缀 | 动作 |
|------|------|
| `收录：` | 创建收录任务（PDF/网页/文档/附件/聊天摘录） |
| `查：` | 查询知识库，要求给依据和限制 |
| `补证：` | 围绕某问题寻找更多来源 |
| `整理：` | 处理待入库资料 |
| `发布：` | 发布到飞书云文档（需确认） |
| `状态：` | 查看任务和知识库状态 |
| `复核：` | 检查某结论是否证据充分 |

无法识别前缀时按普通问题处理；用户明显在提交资料时询问是否收录。

## 超时策略

硬规则见 `RULES.md` 第 14–18 条。以下为操作摘要（两档）：

- ≤10 秒：直接回复。
- >10 秒：先写 `knowledge/tasks/`，**第一条用户消息 = 任务回执**，再后台处理；禁止阻塞频道且无回执。

**顺序（强制）：** 判断耗时 → 写任务文件（若需要）→ **第一条用户消息 = 回执** → 再调工具。

## 收录回复格式

```text
已创建任务：ingest-YYYYMMDD-NNN
领域：...
来源类型：...
来源评分：待评 / strong / usable / weak / rejected
处理策略：...
下一步：...
需要确认：...
```

## 查询回复格式

```text
问题界定：
结论等级：
核心回答：
依据链：
限制与反例：
缺口与下一步：
```

飞书回复可以短，但不能省略依据和限制。长回答先给摘要。

## 任务生命周期

```text
received → classified → processing → drafted → review_needed | auto_applied → done | failed
```

## 发布边界

飞书云文档只保存专题报告、综合页、状态报告。不保存每张依据卡、原始转换产物或高频小文件。
