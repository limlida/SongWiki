# Book-to-EPUB Reviewer 审核模板

## 使命

独立审核 Process 输出，不信任任何判断，全部亲自验证。所有标准从 Process 产出的 TOC 动态获取，不硬编码。

## 输入

- MinerU 原始输出（基准文件）
- Process 输出（待审核文件）
- Process 产出的 `QUALITY_REPORT.md`

## 审核清单（12 项，全部通过才算 PASS）

| # | 检查项 | 方法 | 通过标准 |
|----|--------|------|---------|
| 1 | 图片完整性 🔴 | `grep -c '!\[.*\](' 对比两个文件 | Process >= MinerU（一票否决） |
| 2 | 标题层级 | 计数 H1/H2/H3 | H1=1（书名），H2/H3 数量与 TOC 一致 |
| 3 | 标题顺序 | 逐行读取 H2/H3 | 与 TOC 顺序一致 |
| 4 | 目录保留 | 核对 TOC 区域 | 目录页标题层级与正文一致，无丢失 |
| 5 | LaTeX 残留 | `grep '\$'` | =0（化学式已转为 Unicode） |
| 6 | 图片切断修复 | 随机抽检 3 处图片位置 | 图片前后句子完整连贯。排除：图注（含 著/款/刻/图 及器皿名）、工艺步骤名 |
| 7 | 逐字换行修复 | 全文搜索单行 1-2 汉字 | =0。排除：TOC 条目、章节标题、术语 |
| 8 | NEEDS_CAPTION | 搜索 | =0 |
| 9 | 噪音标题 | 纯数字 `###`、纯英文 `##` | =0 |
| 10 | 缩进完整 | 随机抽样 5 段正文 | 段首有全角空格 |
| 11 | 内容完整 | 字符数对比 | 偏差 ≤ ±10% |
| 12 | 质量报告 | 检查 QUALITY_REPORT.md | 存在且【待确认】项 ≤ 3 |

## 输出格式

```markdown
# Reviewer 审核报告

## 结论: PASS / FAIL

## 逐项结果

| # | 检查项 | 结果 | 详情 |
|---|--------|:---:|------|
| 1 | 图片完整性 | ✅/❌ | MinerU=N, Process=M |
| 2 | 标题层级 | ✅/❌ | H1=1, H2=N, H3=M |
| 3 | 标题顺序 | ✅/❌ | 与 TOC 一致/不一致 |
| 4 | 目录保留 | ✅/❌ | |
| 5 | LaTeX 残留 | ✅/❌ | N 处 |
| 6 | 图片切断修复 | ✅/❌ | 抽检 N 处 |
| 7 | 逐字换行修复 | ✅/❌ | N 处残留 |
| 8 | NEEDS_CAPTION | ✅/❌ | N 处 |
| 9 | 噪音标题 | ✅/❌ | N 处 |
| 10 | 缩进完整 | ✅/❌ | 抽检 N 段 |
| 11 | 内容完整 | ✅/❌ | 偏差 N% |
| 12 | 质量报告 | ✅/❌ | 待确认 N 项 |

## 问题列表（如有）
...

## 建议修复
（Process 应如何修复每个问题）
```

## 通过标准

- 图片 ❌ → 直接 FAIL
- 其他项 ❌ ≥ 3 → FAIL
- FAIL → 回退 Process（最多 3 轮）
