# Reorder

## Overview

将长文档整理为结构清晰、排版统一、便于阅读与维护的 Markdown 文档,严格保持内容信息量与字数基本一致(不擅自精简)。

## Hard Constraints(不可违背)

- 不擅自精简:不得删除信息、概括缩写、减少细节/例子/步骤;允许因标题/序号/少量过渡语带来轻微增减,但不得明显缩短。
- 不改变核心观点、原意与情绪:不得引入用户未提供的新知识/观点/结论;不得扭转语气立场;不得做事实纠错、反驳、劝和或"降级情绪"(例如把强烈指责改成温和建议)。允许在不改变核心观点与情绪强度的前提下,对待编辑文本做更积极的润色与组织:可重写句子、调整语序与段落顺序、拆分/合并句子、补全省略成分、统一术语/人称、添加不改变含义的衔接词;但不得把多句合并成更短表达导致信息量下降,整理后总字数仍需与原文基本一致。
- 以"重排 + 排版"为核心:建立清晰层级结构,统一全篇格式。
- 可追溯:保留原文措辞与段落内容;如有调整,确保在新结构中可轻易找到原信息。

## Workflow(必须按顺序)

### A. 通读与结构诊断
- 快速扫描全文，了解书籍结构（章/节分布）和内容特征。
- 识别 MinerU 遗留的 `<details><summary>natural_image/seal/...</summary>...</details>` 块，标记为需清除。
- 标注明显排版问题：标题层级混乱、图片切断句子、逐字换行、TOC 条目混入正文开头等。

### B. 保留原书目录结构
- 目录页是原书的一部分，**原样保留**：包含页码数字、换行、层级标记，不做任何修改。
- 从目录页提取结构化 TOC，用于后续核对正文标题一致性。
- 确认正文的实际标题层级：原书通常 篇名=H1、节名=H2。

### C. 内容修复与衔接
- **图片切断修复（重排核心）**：通读全文，当图片夹在句子中间切断词句时：先将图片前后的文本碎片合并为完整句子，再将图片块（图片+caption）移至句号或段落自然停顿之后。
- **短行合并**：连续两行且上一行以中文字符结尾、下一行以中文字符开头、总字数均 <30 → 合并为一行。修复 OCR 硬换行导致的短语断裂（如「同时\n存在」→「同时存在」）。
- 图片标题标记：图片下方 <80 字、无句号的短文本 → 合并标记为 `<span class="caption">文本</span>`。
- 出现重复内容:默认不删除;将重复段落放在同一处,并用"(重复信息保留)"提示。
- 不确定的修复:保守处理,标注【待确认】;不编造答案。

### D. 排版与可读性优化(不新增内容,仅改呈现)
- 统一标题层级:使用 H1/H2/H3(或等效的"一级/二级/三级标题"),层级不跳跃。
- 统一列表与编号:同类列表保持一致(项目符号或数字序号),避免混用;步骤/流程用编号列表。
- 统一段落:每段一个主题;段落之间留清晰空行。
- 修复"逐字换行/竖排字符":当同一短语被拆成连续多行且每行仅 1-2 个汉字/符号、可无歧义拼接时,将其合并为正常词句(例如 `对\n数\n收\n益\n率` → `对数收益率`)。
- 修复"公式碎片化"(如 MinerU 未出现则跳过):当公式被断成单字符/单 token 多行导致不可读时,在不新增信息前提下重组为可读的 LaTeX 数学公式;行内用 `$...$`、块级用 `$$...$$`。
- 修复"方括号公式块"(如未出现则跳过):当出现用单独一行 `[` 与单独一行 `]` 包裹的数学表达时,将中间内容转为 `$$ ... $$`。
- 修复"括号内 LaTeX 片段"(如未出现则跳过):当括号内出现 LaTeX 命令/符号(如 `\text{}`、`\cdot`、上下标 `_`/`^`)时,改写为正确的行内公式。
- 修复 LaTeX 下划线(如未出现则跳过):`\text{...}` 内含 `_` 时写成 `\_`。
- 规范"行内/块级公式"选择:括号内解释、同一行句子中的短公式、列表项中的嵌入变量一律使用行内 `$...$`;只有当公式作为独立段落/单独成行展示时才使用 `$$...$$`。避免在一句话里混用普通文本 + `$$...$$`。
- 行内公式分隔符:行内数学写成 `$x$`,不要写成 `$ x $`;`$$...$$` 同理不要在分隔符内侧留首尾空格。为减少解析歧义,行内公式与中文/英文相邻时优先两侧留空格(例如"由 $X$ 自动推导")。
- 修复"句内断行的小公式 token"(如未出现则跳过):当一句话被拆成多行且夹杂短公式/符号导致语义断裂时,将其合并回句子,并按需要改为行内公式。
- 修复 OCR 的"重复 token / 运算符断行"(如未出现则跳过):若连续多行出现相同的单字符变量或运算符,视为排版错误,去重合并并放回句子应出现的位置。
- 规范数学字符(如未出现则跳过):将 Unicode 数学字母(如 X、k)统一替换为 ASCII,优先以 LaTeX 形式呈现关键公式。
- 化学式去 LaTeX:`$\mathrm{Fe}_{2}\mathrm{O}_{3}$` 等化学式 → Unicode(`Fe2O3`)。`\mathrm{}`→去掉,`_{n}`→Unicode 下标,`^{n}`→上标,`\cdot`→`·`。简单上下标公式(`$x^{2}$`、`$H_{2}O$`)→ `<sup>`/`<sub>`。
- 清除 MinerU 遗留块：删除 `<details><summary>natural_image</summary>...</details>` 和 `<details><summary>seal</summary>...</details>` 等 MinerU 自动生成的图片描述块。
- 结构化常见信息形态(不新增内容):
  - 结论/要点:用"要点:"或列表呈现
  - 注意事项:用"注意:"或引用块呈现
  - 步骤/流程:用编号列表呈现
  - 示例:用"示例:"区块呈现
- 原文如含代码/配置/命令:保留为 Markdown 代码块,避免重排导致语义变化。

## Output Contract

输出整理后的完整 Markdown 文档。处理过程中的标注、修复记录、待确认项,写入同目录下的 `QUALITY_REPORT.md`。

- 标题层级使用 H1/H2/H3,层级不跳跃。
- 正文开头保留原书目录结构(提取自目录页)。
- 保留全部内容与细节信息;重复信息不删除,标注"(重复信息保留)"。
- 图片标题使用 `<span class="caption">文本</span>` 标记,构建 EPUB 时需注入 `templates/caption.css`(居中、0.85em、灰色)。
- 不确定的修复保守处理,标注【待确认】写入质量报告。

## Interaction Rules(默认直接开始)
- 默认直接整理,不要求用户先补充信息。
- 仅在无法保守归类且会显著影响理解时,使用【待确认】标注(最多 3 条),不要停工等待回复。
- 除非用户明确要求,禁止输出"缩写版/精简版/总结版"。

## Pre-Delivery Checklist(交付前自检)
- 逐段确认:没有删除原文信息;没有把多段合并成更短表达;没有新增原文未提供的观点/结论。
- 观点与情绪确认:没有"纠错/劝和/中和/降级情绪";润色仅为通顺与一致性服务,核心观点与情绪强度保持不变。
- 形式确认:输出为完整 Markdown 文档;标题层级/列表/编号/段落间距统一。
- 内容确认:重复内容未删除(集中放置并标注);不确定归类已用【待确认】说明。
- 可读性确认:已修复明显的"一字一行/竖排字符/断行公式"等排版破碎问题;对不确定的公式重组已保守处理并标注【待确认】。
